Первый способ создания резервной копии БД - это создание дампов базы по таймеру. Для создания SQL дампов можно использовать утилиту pg_dump.
Пример: pg_dump -U postgres postgres > /var/dump-2022-05-22.sql
Эта команда создаст дамп базы postgres от имени пользователя postgres и поместить его в файл pg_dump -U postgres postgres > /var/dump-2022-05-22.sql
Данную строку можно обернуть в sh скрипт и запускать сервисом crone раз в сутки, таким образом у нас будет сохраняться состояние БД раз в сутки и в случае сбоев моэно будет восстановить данные из дампа.
Восстановить базу из дампа можно командой
psql -U postgres postgres < /var/dump-2022-05-22.sql
